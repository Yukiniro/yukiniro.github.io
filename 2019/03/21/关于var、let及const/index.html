关于var、let及const

var let 以及 const的作用都是用来声明变量的，不过其应用场景和副作用都
有所不同。

  • 作用域


    var声明的变量的作用域是当前执行上下文，比如全局作用域或者函数作用域，但是
    没有块级作用域，并且在全局作用域声明的话会自动挂载到全局对象。而let与const
    不会挂载到全局对象，但是它们却有块级作用域

上面代码中a可以正常输出，而b,c会报错，原因就是块级作用域的区别。

在上面这个例子中，本来的需求是输出0, 1, ... 9，但实际上输出的是10, 10...10
因为var声明的变量没有块级作用域，循环体中访问的i是同一个变量。所有在循环做完后i的值便为10,然后输出。
要解决这个问题，可以将循环体做成一个闭包，然后将i传进去。 不过最简单的是将
var替换为let，这样就会有块级作用域，也就是说每个循环体里的变量i都是单独的个体，

  • 声明提升

    JavaScript中的变量以及函数的声明会提升到当前作用域顶端（函数表达式不会
    进行声明，这里不对函数的声明提升进行讨论）。

    这里可以输出a，不过值时undefined，而输出b，c时却会报错，也就是说,
    var会进行申明提升，而let,const不会进行提升。

    可以作为参考

  • 暂时性死区

    在 ECMAScript 2015 中，let 绑定不受变量提升的约束，这意味着 let 声明不会被提升到当前执行上下文的顶部。在块中的变量初始化之前，引用它将会导致 ReferenceError（而使用 var 声明变量则恰恰相反，该变量的值是 undefined ）。这个变量处于从块开始到 let 初始化处理的”暂存死区“之中。
